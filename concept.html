CONCEPT OF LAYERING

As discussed earlier, a network is a set of devices, connected by communication links. The general concept of a computer network that we’ll discuss here, applies to inter-system communication (between 2 different systems), and not intra-system communication (for such communication, we use Operating Systems’ Inter-Process Communication concept)

The main objective of a computer network is to give an illusion to the client and server that they are present on the same system.

Why layering is done?

Layering for a computer network communication is done so as to divide the various functions of the network into different sections so as to facilitate:

• Encapsulation: To combine the functionalities of one layer for easier access.
• Abstraction: It is done for security reasons. Abstraction implies the functionalities of one layer are hidden form the other layers.
• Ease of Testing: Isolation of a fault is easier in layered structure, as it doesn’t require examining the network as a whole.
• Divide & Conquer mechanisms are proven to optimise the system requirements.

THE OSI MODEL

The purpose of the OSI model is to show how to facilitate communication between different systems without requiring changes to the logic of the underlying hardware and software.

The OSI model is a layered framework for the design of network systems that allow communication between all types of computer systems. It consists of seven layers.

In this layered architecture, a message starts from the sender, and reaches the receiver through a physical link (which might be cable or wireless). While the message passes through the physical link, it may cross some intermediate nodes (routers), which generally have only the bottom 3 layers. The designers of the OSI model identified the related functions with related uses, and collected those to discrete groups that became the layers. Each layer defined a family of functions that differ from the other layers. Thus, the designers created an architecture that is both comprehensive and flexible. Most importantly, OSI model allows complete interoperability between otherwise incompatible systems.

The processes on each machine that communicate at a given layer are called peer-to-peer processes. Each layer is connected to the layer above and below through interfaces for efficient communication between the layers.

Each layer adds its own header (and sometimes a trailer), that provides with the information of the functionalities of that particular layer.

The encapsulated packet has a different terminology at different layers.

Q. Let ‘m’ be the message to be transmitted, ‘n’ be the layers that are present in the network. It is given that each layer adds a header of size ‘h’. What percentage of the network is filled with headers? What percentage is filled with message?



Each layer adds a header of size ‘h’. Therefore, total size of all headers = nh
The network contains the message as well as the headers. Hence, percentage of network filled with headers will be
(nh / (nh + m)) * 100
Similarly, percentage of network filled with message = (m / (nh + m)) * 100

Physical Layer:

Physical Layer deals only with the hardware. To be precise it deals with the functionalities and procedures of the physical links and devices and their mechanical and electrical specifications.

• Physical characteristics of the interface and medium
• Representation of bits (Type of Encoding used)
• Data Rate (Description of Bandwidth)
• Synchronization of bits (Synchronization of sender and receiver clocks)
• Line Configuration (Point-to-Point or Multipoint)
• Physical Topology (Mesh, Star, Ring, Bus, Hybrid)
• Transmission Mode (Simplex, Half-Duplex, Full Duplex)

Since, Physical Layer does not form a major portion of the GATE syllabus, only the important concepts of the physical layer will be discussed. For more extensive details about Physical Layer, click here.

Data Link Layer:

The data link layer is responsible for hop-to-hop delivery. It makes sure that the transmissions in the network are reliable.

• Framing: The stream of bits is divided into data units called frames.
• Physical Addressing:MAC addresses are assigned to each intermediate hop, the sender and the receiver for accurate delivery to the intended user.
• Flow Control: It synchronizes the sender and the receiver. It makes sure that the sender does not send more data than the receiver can receive, by proposing a control mechanism not to overwhelm the user.
• Error Control: The data link layer adds reliability to physical layer by adding mechanisms to detect and retransmit damaged or lost frames. It also uses a mechanism to recognise duplicate frames.
• Access Control: It is necessary to determine which device will gain control over the link at any given time.

Network Layer:

The network layer is responsible for source-to-destination delivery of a packet, possibly across multiple networks. The network layer header provides with the IP address of the destination, which generally can be broken down into two parts, Network ID (NID) and Host ID (HID). The NID gives the address of the network where the destination node is present, and the HID gives the address of the node in that network.

• Logical Addressing: Physical Addressing handles the addressing problem locally. If a network boundary, we need the logical addressing system to distinguish the source and destination systems.
• Routing: When independent networks or links are connected to create internetworks, the connecting devices route the packet to the final destination. This process is called routing and is implemented at the network layer.

Transport Layer:

The transport layer is responsible for process-to-process delivery. A process is an application running inside the host. At any time, more than one process may run at a host. Selecting the correct process to which the packet is to be delivered is done using another type of addressing. It also ensures that the whole message arrives intact and in order, overseeing both flow and error control at the source-destination level.

• Service-point addressing: As discussed earlier, a computer might run several processes. The transport layer header is therefore responsible for providing port-address, so that the entire, message gets to the correct process on that computer.
• Segmentation and Reassembly: A message is divided into segments with their corresponding sequence number. These segments may arrive out-of-order at the destination; hence they need to be reassembled at the destination.
• Connection-Control: The transport layer can be either connectionless or connection-oriented.
• Flow Control: Flow control is performed end to end.
• Error Control: Error control is performed process-to-process than at the link level.

Session Layer:

The session layer is the network dialogue controller. It establishes, maintains, and synchronizes the interactions among communicating systems.

• Dialog Control: It allows the communication between 2 processes in either half-duplex or full-duplex mode.
• Synchronization: The session layer allows a process to add checkpoints. These checkpoints are added after regular intervals and acknowledged independently. They are used to facilitate faster crash recovery.

Presentation Layer:

It is concerned with the syntax and semantics of the information exchanged between two systems.

• Translation: The character strings, numbers etc. must be converted to bit streams before being transmitted. Different computers use different encoding schemes.
• Presentation layer is responsible for the interoperability between these different encoding methods
• Encryption: The sender transforms the original message into some other form and sends the resulting message out on the network. Decryption reverses the original process.
• Compression: Data compression reduces the number of bits contained in the information.

Application Layer:

The application layer enables the user, whether human or software, to access the network. In other terms, it acts as an interface between the users and network.

• Network Virtual Terminal: A network virtual terminal is a software version of a physical terminal, and it allows user to log in to a remote host.
• File transfer, access and management: This application allows the user to make changes, access files, and retrieve files on a remote host.
• Mail Services: This application provides basis for e-mail forwarding and storage.
• Directory Services: This application provides distributed data sources and access for global information about various objects and services.

The packet flow in a network happens as follows. The data is generated by the application layer. The generated data passes layer by layer adding the headers of each layer and passes through the network link. It encounters many intermediate routers. It is important to notethat routers consist of only 3 layers, and not the upper layer, since the functionality of a router is only to route the packet to the intended destination (generally using the shortest path) and the upper layers are not required (only logical addressing is necessary). The packet reaches the destination and the headers are removed one by one at each corresponding layer exactly opposite to the way they were added.

THE TCP/IP PROTOCOL SUITE

it is similar to the OSI Model, with a minor difference that the functionalities of the presentation and session layer are merged with that of the application layer and transport layer, and the data link and physical layers are merged into hot-to-network layer. Hence, the TCP/IP protocol suite has 4 layers: host-to-network, internet, transport, application. However, for general purposes we consider the TCP/IP protocol suite to be of 5 layers; namely, physical, data-link, network, transport, application. And if not specified, please consider the network model consisting of these 5 layer, and follow the TCP/IP protocol suite.
