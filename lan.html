LAN TECHNOLOGIES (ETHERNET)

The IEEE 802.3 standard describes Ethernet.

Points to remember for Ethernet



Ethernet Frame Format


The preamble and SFD are used to alert the stations that there is a packet in transit. It aids synchronization. They form the physical layer header. The part from Destination Address (DA) to Length is the DLL header and the CRC (Cyclic Redundancy Check, used for error detection) is the data link trailer.

The MAC SA and DA are of 48 bits each, they are the physical addresses of the source and the first hop respectively. (If the first hop is the router, then DA is the MAC of that router, if it is the destination, the MAC is of the destination) The MAC addresses on the packet change at each router. For example, consider the figure shown below


The source and destination MAC changes as the packet goes from the sender to the first hop router. The source MAC address is assigned the MAC of the router and the destination MAC is assigned the MAC of destination D. Therefore data link layer is necessary for the router for hop-to-hop delivery.

Types of MAC addresses:

• Unicast: To send the packet to a single destination, unicast address is used. Unicast MAC addresses are identified by the Least Significant Bit of the 1st byte of the MAC. If that bit is 0, it is a unicast address.
Example: 24 : 32 : 5F : 00, since 24 can be represented in binary as 0010 0100, it is a unicast address.
• Multicast: Multicast packets are sent to a group of stations. If the LSB of 1st byte is 1, then it is a multicast packet.
Example: 65 : 32 : 5F : 2E, since 65 can be represented as 0110 0101, it is a multicast address.
• Broadcast: Sending the packets to all the hosts in the network. All 1’s represent the broadcast MAC. FF : FF : FF : FF is broadcast MAC address.

As in CSMA/CD,
Tt ≥ 2 * Tp
L ≥ 2 * Tp * Bw
If we substitute the standard Ethernet values in this, we get
L ≥ 64 Bytes
Therefore, we need to send more than 64 Bytes for the Ethernet to work properly. If the application layer sends lesser data than this, the transport layer will wait for accumulating more data until it has ≥ 64 Bytes to send.
On the other side of the coin, a station has to be restricted from sending more than a certain amount of data, otherwise a station which starts sending the data first will keep on sending large packets and will not let other stations send data. This situation is called monopolization.
The max fixed data in Ethernet is set at 1500 Bytes.

Disadvantages of using Ethernet:

• Ethernet cannot be used for real-time applications as they require sending and receiving data of varied sizes.
• Since, it has a limitation on minimum and maximum size of data; it cannot be used for interactive applications.
• There is no priority field in Ethernet frame format, hence all stations are treated equally and it is not suitable for client-server applications
