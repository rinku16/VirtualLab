PHYSICAL TOPOLOGY

It defines the way nodes and links in a network are laid out physically.

1. Mesh Topology:

In this topology, every device has a dedicated point to point duplex link to every other device. Hence, as you might have guessed, mesh topology forms a complete graph.

Q. What is the total number of links in a mesh topology with ‘n’ nodes?

Tip: Think about the solution first, and think about more than one way to solve it. It will be mentioned specifically if a problem has more than one way to reach a solution.


Approach 1- Every node is connected with every other node; think about it as a way of selecting any two nodes, and having a link between them. Therefore,

No. of links = nC2 = n(n-1) / 2

[Remember, nCr = (n)! / (n-r)!(r)!]

Approach 2- Think about a party in which everybody shakes hands with everyone else, but only once. Now evaluate the total number of handshakes. If we consider the first person, he has to shake hands with n-1 others, the next person will have to shake hands with n-2 others, since he already shook hands with the first and so on.

Therefore, total number of handshakes (total number of links as per analogy) will be,

(n-1) + (n-2) + (n-3) + (n-4)………………+2 + 1 = n(n-1) / 2

*Remember, 1 + 2 + 3 + 4……….n = n(n+1) / 2, replace n by n-1. (Arithmetic Progression)]


Advantages of Mesh Topology:

• Traffic problems are eliminated, since every 2 nodes have a dedicated point to point link.
• Mesh topology is robust. If one link becomes unusable, it does not affect the entire system.
• There is privacy and security, since only the intended receiver is able to see the data.
• Point to point links makes fault isolation and fault identification easy.

Disadvantages of Mesh Topology:

• Amount of cabling required is quite high. As we’ve seen n(n-1) / 2 dedicated links are required, which, in practice is a very high number.
• Number of I/O ports required is high as well.
• Installation and Reconnection are difficult.
• Hence, mesh topology becomes really expensive owing to the large number of hardware devices required to set it up.

Therefore, mesh topology is used only in limited applications, generally implementation of LAN networks as they have less number of nodes and span less area.

2. Star Topology:

In this topology, every device has a dedicated point to point link to only a central system, called hub. The hub acts as a controller for exchange of data between the devices. If any device wants to send data to any other device the data goes through the hub to the receiving device.

Total number of links in star topology with ‘n’ nodes = n (One link to hub, for each node)

Advantages of Star Topology:

• Less expensive than mesh topology. Far less cabling is required in star topology.
• Addition and deletion of a connection involve only one link, between device and hub.
• Robustness.
• Easy fault identification.

Advantages of Star Topology:

• Complete dependency on a single point, i.e. hub. If the hub goes down, the entire system goes down.
• Number of links required is still more than some other topologies.

3. Bus Topology:

Bus topology, unlike above mentioned topologies, is multipoint connection. A long cable acts as a backbone, in bus topology, to link all other nodes in the network. Nodes are connected to the bus by drop lines or taps. A drop line is a connection between the main cable and the device. A tap is a connector that splices into the main cable to create contact with the metallic core.

Advantages of Bus Topology:

• Ease of installation. It uses less cabling than star or mesh topology. The backbone cable stretches as far as the entire area, and the drop line has to reach only as far as the nearest point on the backbone cable.

Disadvantages of Bus Topology:

• Difficult reconnection and fault-isolation. The distances in the bus topology are often chosen to optimise the requirements. Hence, it is difficult to add new devices.
• Fault or break in the backbone bus cable, stops all the transmissions.

4. Ring Topology:

As the name suggests, each device has a dedicated point to point link with 2 devices on either side of it, just like a ring (Recall that a cycle in a graph has degree 2 for each vertex). A signal is passed in one direction, from device to device until it reaches its final intended receiver. As the signal attenuates as it travels, each device is incorporated with a repeater for regenerating the signal to compensate for the loss of energy.

Advantages of Ring Topology:

• Easy to install and reconfigure. Addition or deletion of a device requires changing of only 2 connections.
• Fault isolation is easy. If one device does not receive the signal within its intended period of time, it can issue an alarm to alert the network operator.

Disadvantages of Bus Topology:

• Unidirectional traffic may lead to starvation.
• A break in the ring can disable the entire network. (solved by using a dual ring)

5. Hybrid Topology:

Using more than one topology to optimise the requirement is known as hybrid topology.
